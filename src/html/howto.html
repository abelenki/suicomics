  • What is a required item on a page?

    • A required item is a virtual goods designed by a creator for his/her book
    • It has a name, a small picture, a price and a short description
    • Creators design these items on the book edit page
    • It can be placed on a particular page in the Required Items section
    • A required item means that the reader must have it before reading that page
    • If the reader does not have it in her inventory, she has to buy it
    • A required item usually cannot be placed in the reader's gallery
    • There can be multiple items required on a page, but only one is necessary
    • A reader can choose to buy at least one of the items to go on reading

  • What is a reward on a page?

    • A reward is a virtual goods designed by a creator for his/her book
    • It has a name, a small picture, a price and a short description
    • Creators design these items on the book edit page
    • Add a virtual goods to a particular page as reward for all readers
    • A reward item can be collected by a reader only once
    • And it can be placed in readers' galleries if the creator so defines
    • A reward item is displayed in a way determined by a dynamic control
    • It can show up automatically soon after the reader turns to the page
    • Or it can be displayed as a button or image for readers to click and pick

  • What is a quest and how to create it?

    A quest is a task for the reader to complete. It is designed by the creator for the background story. It consists of a big image as the scene and a list of items placed on it. Originally the reader has none of these items, and while the reader reads the book, he/she can collect the items one by one, and eventually collect all items to complete the quest. When a reader completes a quest, he/she may get a prize.

    • Create an image of size 740x500 pixels and save as JPG file
    • Upload it as a virtual goods with display=false, price=0
    • Create a number of small transparent images as PNG files
    • Stick these small images onto the big one to find out top-left coordinates
    • Upload these stickers as virtual goods and set a price if for sale
    • Create a quest on the book edit page
    • Each quest has a name, a background image and up to five stickers
    • Give each sticker an x and y coordinate, scale of 0.1 to 1.0
    • And assign another virtual goods as a reward for completing the quest
    • The reward and stickers can be any virtual goods either bought or collected

  • Creat a quest script

    A quest script is in JSON format. If you're not sure what JSON is, you could either contact me for help with your quests, or wait until our visual quest editor is out.

    • A book can have multiple (10) quests, as a list of objects: [{...},{...},...]
    • Each quest has a "qid" which is an image ID, "qname" as the name of the quest, and a list of "items", a prize and an introduction.
    • Items are the shadow objects in the quest picture, these are virtual goods placed throughout the book pages.
    • An item has a "vgid" ID, x and y coordinates, "sc" as a scaling factor, and possible filters (to be available).
    • A prize is also a virtual goods image with an ID given here. If no prize, set the ID as zero
    • You can find the image IDs from the virtual goods table on the Creators page.
    • The quest image, the background picture, must be of 740x500 pixels in size, objects should be smaller.
    • Object images should have a transparent background saved as PNG files.
    • It's better to avoid objects overlapping each other.
    • It's recommended to design the objects in different layers, locate the positions and design the script.
    • Quest script is validated upon submission and will be rejected if invalid.
    • You can validate any JSON text on http://www.jsonlint.com/ website.
